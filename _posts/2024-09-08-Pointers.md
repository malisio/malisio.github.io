## What are Pointers? 
- Pointers are **derived** variables just like other data types but the main **difference** is that they hold memory addresses not values.
Example of a simple pointer:

```c
int k = 2; 
int *ptr =  &k;
```
- Notice to declare a pointer we use `*` , and to get the memory address of the variable k we use the `&` operator. Why would we want a memory address instead of a value in the first place? First let think of memory like being a street it is made of blocks (houses), each house holds a value (person) and the same houses have an address (memory address), obviously you send your address not your house to your friend no?
```c 
#include <stdio.h>

int main(){

	int k = 2;
	int *ptr = &k;
	printf("Memory address of k:%p\nValue of k %d\n",&k,k);
	printf("Value of ptr: %p\nValue after deferencing %d\n",ptr,*ptr);
	return 0;
}
/* Sample Output:
Memory address of k:0x7ffecd8fc4bc
Value of k 2
Value of ptr: 0x7ffecd8fc4bc
Value after dereferencing 2
*/
```
- As expected they share the same memory address, and the same value,one new thing is that to access a the original value pointer by the pointer we need to deference it first we can use the `*` to do so.
- Pointers Do have memory address to to access them we can use `&`  example:  
```c
#include <stdio.h>

int main(){
	int k = 2; 
	int *ptr = &k;
	printf("Memory address of k: %p\n",&k);
	printf("Memory address of ptr: %p\n",&ptr);
	return 0;
}
/*Sample Output:
Memory address of k: 0x7ffdb8c7749c
Memory address of ptr: 0x7ffdb8c774a0

```
- Memory addresses are displayed in hex that why we use `%p` also pointers size are larger than other data types we can access the size of a data type using `siezof();` to display the size on the current architecture.

## Pointers arithmetic 
- For example let us consider a block of memory containing 5 int elements with the address of 100 (decimal) that is 20 of bytes set aside by the compiler, Now we want our pointer variable to point at the second integer (since pointers point by default to the first element). We can just add number 1 to the ptr xD. `ptr++` and since the compiler knows this is a pointer (memory address) it is going to look into the next memory address and assign it to it, small note it doesn't add 1 it adds the size of the integer xD. 
## Arrays and pointers 
- Arrays and pointers tend to have an interesting relationship between each other since an array doesn't directly hold the value but rather points also to a value that means that doing such thing as below would be legal.
```c 
#include <stdio.h>
int array[6]={1,2,3,4,5,6};
int *ptr=NULL;
int main(){
	ptr = array;
	printf("Array poiting at: %p ptr poiting at:%p\n",array,ptr);
	return 0;
}
/*sample output
Array poiting at: 0x64a6d635f020
ptr poiting at:0x64a6d635f020
*/
```
- Notice how they're pointing at the same memory address which means we can do something like this: 
```c 
#include <stdio.h>
int array[6]={1,2,3,4,5,6};
int *ptr=NULL;
int main(){
	ptr = array;

	for(int i = 0 ; i < 6; i++){
		printf("array[%d] = %d\n",i,array[i]);
		printf("ptr:+ %d =  %d\n",i, (*ptr)++;
		}

}
/* Sample out put
```

 Let discuss why `ptr= NULL;` :
 1) It is good practice to make a pointer point to nothing (NULL) when initialized specially if it isn't going to be used.
 2) To avoid undefined behavior or weird behavior. 
 3) Also assigning null to pointer means that something went wrong.
 Let talk about this `printf("ptr:+ %d =  %d\n",i, (*ptr)++;`  what exactly does `(*ptr)++` Do? 
 First it deference the ptr then jumps to the next memory address there no difference between `(*ptr)++` and `*ptr++` or even `*(ptr)+i` try playing around with the code and you try predicting the output each time(they well be the same).   
 Let get back to out topic arrays and pointers have almost the same behavior. 
 However they are similar but **NOT** the same: 
 We can do `ptr = array` but cannot `array = ptr` The reason behind it. while ptr is a variable and dynamic array is constant. That means when the first element of the array `array[0]` is assigned it cannot be changed.
 Other writers tend to name arrays constant pointers. Hence let try to understand what they mean by "constant" in that term,Looking back at the variable definition When we declare a variable we tell the compiler to set a side a block of memory in the ram to be used by that specific variable which can be accessed in two ways either being an lvalue or rvalue (It doesn't make sense but lvalue rvalue you well understand from this example)
 ```c
 int k = 2; 
```
k is a left value the value that well receive the rvalue and 2 is the rvalue.
Back the the topic when the variable used on the left side the compiler interprets as memory location to move to that value resulting from evaluation of the right side of the assignment operator. But, when used on the right side of the assignment operator, the name of a variable is interpreted to mean the contents stored at that memory address set aside to hold the value of that variable.
From that since array is constant once the compiler establishes where the array it self is to be stored and on seeing:
